# 系统说明文档

## 1. 项目背景

### 1.1 研究与开发动因
教育心理学及学校心理支持相关知识广泛分布于教材、政策文件、教师手册、培训资料、研究论文和实践报告之中。对于学生、家长和一线教师而言，这些资料虽然具有较高参考价值，但在实际使用过程中存在获取效率低、检索路径分散、专业表达门槛较高等问题。

在真实场景中，用户往往不是以标准学术术语提出问题，而是以自然语言描述自身处境。例如，学生可能会询问“考试前总是失眠怎么办”，家长可能会询问“总催孩子学习是不是会让他更焦虑”，教师则可能更关注“发现学生情绪异常后该如何支持”。此类问题具有明显的情境性、综合性和语义模糊性，传统检索方式很难稳定提供高质量支持。

### 1.2 需要解决的问题
本系统主要试图解决以下三类问题：
- 教育心理学资料来源多样、格式不统一，缺乏统一知识组织方式；
- 用户问题往往以口语化、情境化方式表达，传统关键词检索命中率有限；
- 纯大语言模型虽然能够生成较流畅的答案，但若缺乏知识库约束，容易出现事实漂移、证据不足或不恰当扩展。

### 1.3 采用 RAG 的原因
相较于传统搜索方法，RAG（Retrieval-Augmented Generation，检索增强生成）能够同时利用语义检索能力和语言生成能力，在教育心理学场景中具有更高适配性。其优势主要体现在：
- 能够基于语义相似性召回与问题相关的知识片段，而不仅限于关键词完全匹配；
- 能够在知识证据基础上生成结构化、可读性更强的回答；
- 能够通过引用检索结果降低模型幻觉风险，提高回答的可解释性和可信度。

因此，本项目采用 RAG 作为核心技术路线，用以构建一个面向教育心理学场景的知识问答系统。

## 2. 系统目标

本系统的总体目标包括以下四个方面：
- 构建教育心理学与学校心理支持领域的专用知识库；
- 支持基于自然语言的问题输入与知识检索；
- 生成具有准确性、可解释性和证据支撑的回答；
- 服务学生、教师、家长等典型教育场景中的心理支持与教育咨询需求。

## 3. 系统整体架构

### 3.1 总体流程
系统整体工作流程如下：

`用户输入问题 → 查询向量化 → 向量检索 Top-K 文档 → 上下文拼接 → 大模型生成答案 → 输出带证据回答`

### 3.2 架构示意图

```text
+------------------+
|      用户        |
+------------------+
          |
          v
+------------------+
| 自然语言问题输入 |
+------------------+
          |
          v
+------------------+
| 查询向量化模块   |
+------------------+
          |
          v
+------------------+
| 向量数据库检索   |
|   Chroma Store   |
+------------------+

+------------------+

+------------------+

+------------------+

+------------------+

+------------------+
```

### 3.3 主要组成模块
- 前端交互层：基于 Streamlit 实现，负责文件上传、知识导入、检索问答与结果展示；
- 后端服务层：基于 Python 编写，负责数据导入、向量检索、上下文组装与答案生成；
- 向量存储层：采用 Chroma 保存持久化向量索引；
- 知识层：包括原始资料层、结构化知识层与向量化检索层。

## 4. 数据构建

### 4.1 数据来源
本系统的数据来源主要包括以下几类：
- 教育心理学教材；
- 学生心理健康、家校沟通、学校支持等相关政策文件；
- 教师教学手册与学校实务资料；
- 学术论文及研究综述；
- 国际组织报告，如 WHO、OECD 等教育与心理健康相关资料。

在当前版本中，系统已经接入：
- 一份教育心理学与学校心理支持结构化知识包；
- 一份大学教师教学手册；
- 一份教育心理学教材 PDF 文档。

### 4.2 数据处理流程
数据处理的基本流程如下：

`PDF / TXT / 原始文档 → 文本抽取 → 清洗规范化 → 分块 → metadata 标注 → JSONL 结构化 → 向量化入库`

其中：
- 对于已具备清晰字段结构的知识包，系统直接解析为结构化 JSONL；
- 对于 PDF 文档，系统先进行页级文本抽取，再生成结构化记录；
- 所有清洗后的数据统一保存在 `data/processed/` 目录中。

### 4.3 分块策略
分块策略是 RAG 系统效果的关键环节之一。本项目在分块设计上遵循以下原则：
- 每个 chunk 尽量只表达一个相对完整的语义单元；
- chunk 长度保持适中，避免过短导致上下文不足，也避免过长导致语义混杂；
- 为每个 chunk 添加结构化 metadata，以支持后续检索过滤、风险控制和证据展示。

推荐的分块原则包括：
- 长度控制在约 300–600 tokens 左右；
- 尽可能按语义单元而不是机械长度切分；
- 增加主题、关键词、用户对象、风险等级等标签。

当前系统中：
- 结构化知识包已预先分块；
- 通用文本导入时使用可配置长度切分；
- PDF 当前采用页级切分，后续可进一步细化为章节级或主题级切分。

## 5. 向量化与存储

### 5.1 向量模型选择
对于中文教育心理学场景而言，向量模型需要具备较强的中文语义理解能力。

在课程设计或论文写作中，可推荐使用：
- `bge-large-zh`

在当前实现中，系统采用可配置的 OpenAI Compatible Embedding 接口，默认模型可通过 `.env` 文件设定。当前实现默认使用 `text-embedding-3-small`，但保留替换空间，后续可根据实验需求切换为更适合中文的专用向量模型。

### 5.2 向量数据库选择
本项目当前采用 Chroma 作为向量数据库。

选择理由如下：
- 本地持久化部署方便，适合课程项目与原型验证；
- Python 集成难度较低；
- 对中小规模知识库场景具有较高实现效率；
- 有利于快速构建、测试和展示完整 RAG 流程。

后续可替换或扩展为：
- FAISS
- Milvus
- Elasticsearch 向量检索方案

## 6. 检索机制

### 6.1 当前检索方式
当前系统采用基于向量相似度的 Top-K 检索机制。基本流程如下：
1. 对用户问题进行向量化；
2. 在 Chroma 中执行向量相似度检索；
3. 返回最相关的 Top-K 知识片段；
4. 将这些片段作为上下文送入大模型生成模块。

### 6.2 可扩展检索优化
为进一步提升检索效果，系统后续可以引入以下机制：
- Hybrid Search：结合关键词检索与向量检索；
- Query Expansion：对用户问题进行扩展或改写；
- Question Routing：对问题进行类别分流，如理论类、应用类、风险类；
- Reranker：对召回结果进行二次重排序。

对于教育心理学场景而言，这些优化尤为重要，因为用户表达往往较口语化、情境化，且高风险问题需要更稳健的召回策略。

## 7. 生成模块

### 7.1 生成逻辑
生成模块负责在检索证据基础上组织最终回答。其输入主要包括：
- 用户问题；
- 检索到的 Top-K 相关文档；
- 系统级提示词（System Prompt）。

系统最终输出的回答要求：
- 内容基于知识库证据；
- 语言清晰、专业、支持性强；
- 对高风险问题保持边界意识；
- 在适用情况下提供引用标记。

### 7.2 Prompt 设计
Prompt 设计是本项目的重要加分点之一。当前生成模块特别强调：
- 只依据检索到的资料作答；
- 不编造事实，不随意扩展为诊断性判断；
- 优先提供教育心理学视角下的解释与行动建议；
- 在涉及自伤、自杀、严重情绪危机等情境时，必须明确建议联系监护人、学校心理老师或专业机构；
- 保持温和、专业、可执行的回答风格。

这一设计与本项目的领域目标一致，即在提供知识支持的同时，避免不恰当风险输出。

## 8. 系统实现

### 8.1 编程语言
- Python

### 8.2 主要框架与工具
- Streamlit：前端交互界面
- FastAPI：后端 API 服务
- Chroma：向量数据库
- OpenAI Compatible API：Embedding 与生成能力
- `pypdf`：PDF 文本抽取

### 8.3 实现方式
本项目采用轻量级自定义 RAG 实现，而非完全依赖大型框架封装。这样做的好处在于：
- 系统结构更透明，便于课程展示与评审说明；
- 更容易针对教育心理学场景加入安全边界与领域规则；
- 更便于后续局部替换或模块化优化。

├── data/
│   ├── raw_sources/

## 9. 应用场景

### 9.1 面向学生
- 考试焦虑支持
- 学习动机与拖延问题
- 情绪调节与校园适应

### 9.2 面向教师
- 教学策略检索
- 课堂管理与反馈方式建议
- 学生风险识别与支持边界说明

### 9.3 面向家长
- 家校沟通建议
- 降低催促与压力型互动
- 支持孩子学习与情绪发展的方式

### 9.4 面向学校支持体系
- 学校心理支持流程设计
- 心理筛查边界与转介机制说明
- 学校、家庭、社会资源协同支持

## 10. 总结
本项目构建了一个面向教育心理学场景的 RAG 知识库系统。系统通过整合结构化知识、原始文献、语义检索和大模型生成能力，使教育心理学资料能够以更自然、更可解释的方式服务于学生、家长和教师。

该系统的价值不仅在于实现了一套完整的技术流程，更在于其将领域知识、安全边界和真实教育情境结合起来，为教育心理支持型问答系统提供了较为完整的原型基础。
